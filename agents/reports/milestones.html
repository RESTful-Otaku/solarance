§Where we are right now

CURRENT FOCUS
M2 is done — M3 (Two-Faction MVP Setup) is up next. M2 closed out with the #101 messaging redesign (PR #143): ServerMessage/ServerMessageRecipient replaced by five Channel tables + DirectServerMessage, the codebase's first STDB Views (a preview of M7's anti-cheat work), and login-relative read state. Design rationale lives in docs/adr/0001-messaging-redesign.md. The playtest surfaced three new issues — #141 (mining desync, M5), #144 (crate pickup UI, M5), #145 (server-admin client, M4) — all triaged into later milestones. M3 has no started issues yet.

The seven MVP milestones below replaced the original ten-milestone plan (now archived). Each one's exit gate is the only check that matters — the issue list is just the path to get there.

§Post-MVP Review

⬢ Gated, intentionally empty

The Post-MVP Review milestone holds zero issues by design. It unlocks only after MVP ships AND M7 (Anti-Cheat Hardening) completes. The gate is MVP success criterion 5: the player wants to log in again tomorrow.

Once it passes: re-read the Future Vision section of the MVP design doc, run a real retro on what the gap analysis missed, and pick the single Future Vision v1.0 item with the highest ratio of "amplifies the existing loop" to "build cost". Build only that.
